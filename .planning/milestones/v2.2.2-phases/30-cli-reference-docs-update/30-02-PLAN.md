3. **Globset Matching** — Document that file pattern matching uses the `globset` crate instead of naive contains(). Patterns in `files:` fields support full glob syntax (e.g., `**/*.rs`, `src/**`). Mention `build_glob_set()`.

4. **External Logging Fields** — Document the `logging:` section in hooks.yaml (if it exists at the settings level) with backends: OTLP, Datadog, Splunk. Show configuration fields for each backend (endpoint URL, API key env var, etc.). Read `rulez/src/config.rs` for LoggingConfig struct.

5. **tool_input Fields in Eval Context** — Document that `build_eval_context()` injects `tool_input_` prefixed variables from the tool invocation's input object into the `enabled_when` evaluation context. E.g., `tool_input_command` for a Bash tool call. Note: evalexpr treats Float(30.0) != Int(30).

6. **Regex Fail-Closed** — Document that invalid regex patterns in matchers cause the rule to NOT match (fail-closed), rather than crashing or matching everything. Mention `get_or_compile_regex()`.

Keep existing schema field documentation. Only add/update sections for new features.
