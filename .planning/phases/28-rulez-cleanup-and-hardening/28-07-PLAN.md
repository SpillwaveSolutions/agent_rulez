**Option A — Component-level debounce (preferred if filterText is set directly from input):**

```tsx
import { useState, useEffect } from 'react';

// Inside component:
const [inputText, setInputText] = useState('');
const { setFilterText } = useLogStore();

// Debounce: only call setFilterText after 200ms of no typing
useEffect(() => {
    const timer = setTimeout(() => {
        setFilterText(inputText);
    }, 200);
    return () => clearTimeout(timer);
}, [inputText, setFilterText]);

// In the input element:
// onChange={(e) => setInputText(e.target.value)}
// value={inputText}
```

**Option B — Store-level debounce timer (if filtering happens inside the store action):**
In `logStore.ts`, add a debounce timer to the `setFilterText` action:
```typescript
let filterDebounceTimer: ReturnType<typeof setTimeout> | null = null;

// Inside the store:
setFilterText: (text: string) => {
    // Update the input display value immediately (no flicker)
    set({ filterText: text });

    // Debounce the expensive applyClientFilters() call
    if (filterDebounceTimer) clearTimeout(filterDebounceTimer);
    filterDebounceTimer = setTimeout(() => {
        get().applyClientFilters();  // Only run heavy filtering after 200ms pause
    }, 200);
},
```

Choose the approach that fits the existing architecture (from Task 1 findings). If `applyClientFilters()` is called inside `setFilterText`, Option B is more appropriate. If it's reactive (called via useEffect watching filterText), Option A works better.
