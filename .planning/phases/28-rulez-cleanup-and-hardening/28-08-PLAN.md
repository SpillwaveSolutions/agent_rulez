Based on Task 1 findings, implement parallel evaluation. The safest approach that preserves correctness:

**Phase 1 — Parallel matching (stateless, safe to parallelize):**
```rust
// Collect all rules that match the event in parallel
use futures::future::join_all;

const PARALLEL_THRESHOLD: usize = 10;

if config.enabled_rules_count() < PARALLEL_THRESHOLD {
    // Use existing sequential path for small rule sets
    return evaluate_rules_sequential(config, event, debug_config).await;
}

// Parallel matching phase: run rule_matches() concurrently
let match_futures: Vec<_> = config.enabled_rules().iter().map(|rule| {
    let rule = rule.clone();  // Rule must impl Clone
    let event = event.clone(); // Event must impl Clone
    async move {
        let matches = matches_rule_for_parallel(&rule, &event);
        (rule, matches)
    }
}).collect();

let match_results = join_all(match_futures).await;
let matched_rules: Vec<_> = match_results.into_iter()
    .filter(|(_, matches)| *matches)
    .map(|(rule, _)| rule)
    .collect();
```

**Phase 2 — Sequential action execution (preserves priority/merge semantics):**
```rust
// Execute actions for matched rules in priority order (sequential to preserve semantics)
let mut responses = Vec::new();
for rule in &matched_rules {
    let resp = execute_rule_actions_with_mode(rule, event, debug_config).await?;
    responses.push(resp);
}

// Merge all responses
merge_responses_with_mode(responses, debug_config)
```

Key constraints:
- `Rule` must be `Clone` (check models.rs — add `#[derive(Clone)]` to Rule if missing)
- `Event` must be `Clone` (check models.rs — add `#[derive(Clone)]` to Event if missing)
- The parallel threshold is conservative — only applies when there are 10+ enabled rules
- The matching phase is pure (no side effects) so safe to parallelize
- Action execution remains sequential to preserve `merge_responses_with_mode()` semantics

If `Rule` or `Event` cannot be cloned (e.g., contains non-Clone types), use `Arc<Rule>` instead:
```rust
use std::sync::Arc;
let rules: Vec<Arc<Rule>> = config.enabled_rules().iter().map(Arc::new).collect();
```
