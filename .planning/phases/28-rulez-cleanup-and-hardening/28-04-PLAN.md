Read debug.rs fully. Understand the flow:
1. `run_json_mode()` loops over `config.rules`, calls `rule_matches_event(rule, &event)` for each rule
2. It separately calls `hooks::process_event(event, &debug_config)` to get the final response
3. The per-rule `JsonRuleEvaluation` struct has `details: Option<String>` — currently set to `Some("Rule matched")` or `Some("No match")`

**Approach:** After `hooks::process_event()` returns the response, extract run script information from the response and enrich the matched rules' traces.

Add a `script_output` field to `JsonRuleEvaluation`:
```rust
#[derive(serde::Serialize)]
struct JsonRuleEvaluation {
    rule_name: String,
    matched: bool,
    time_ms: f64,
    #[serde(skip_serializing_if = "Option::is_none")]
    details: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    pattern: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    input: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    script_output: Option<String>,  // Add this field
}
```

After `hooks::process_event()` returns, check if the response contains injected context or a block reason — these are the outputs of `run:` scripts that inject or block. Update the matched rules' evaluations:

```rust
// Enrich matched rule evaluations with action results from process_event
// The response.context contains injected content (from run scripts that inject)
// The response.reason contains block reason (from run scripts that block)
for eval in &mut evaluations {
    if eval.matched {
        let action_info = if !response.continue_ {
            if let Some(ref reason) = response.reason {
                Some(format!("[DEBUG] Script result: blocked — {}", reason))
            } else {
                Some("[DEBUG] Script result: blocked".to_string())
            }
        } else if let Some(ref context) = response.context {
            Some(format!("[DEBUG] Script result: injected {} chars", context.len()))
        } else {
            Some("[DEBUG] No run script or script produced no output".to_string())
        };
        eval.script_output = action_info;
    }
}
```

Also update the human-readable mode (`run()` function) to print a note when matched rules have run scripts. After the response is printed, check `config.rules` for rules with `actions.run` (script_path) and print:
```
[DEBUG] Run scripts were exercised via hooks::process_event() above
```

Note: The per-rule attribution is approximate since `hooks::process_event()` runs all matched rules together and returns one merged response. This is a best-effort trace improvement, not per-script output capture. Comment this clearly in the code.
