  <files>cch_cli/src/adapters/mod.rs, cch_cli/src/adapters/gemini.rs</files>
  <action>
Create a Gemini adapter module that:
- Defines serde structs for Gemini hook input (event name, tool name, tool input, session metadata) and output schema (decision/allow/deny with reason/systemMessage).
- Maps Gemini tool events to RuleZ EventType (BeforeTool -> PreToolUse, AfterTool -> PostToolUse) and normalizes tool_input fields (file path, content, new_string) into the keys RuleZ matchers already expect.
- Maps non-tool lifecycle events to the closest existing EventType (SessionStart/SessionEnd/Notification/PreCompact) and preserves raw Gemini payload in tool_input for future rules.
- Includes a response translator from RuleZ Response -> Gemini response with strict JSON (no stdout noise) and tests that cover mapping and normalization.
  </action>
  <verify>cargo test -p cch_cli gemini</verify>
  <done>Gemini input parses without panics, event mapping produces RuleZ Event with normalized tool_input, and response mapping emits valid Gemini JSON.</done>

  <files>cch_cli/src/cli/gemini_hook.rs, cch_cli/src/cli.rs, cch_cli/src/main.rs</files>
  <action>
Add a CLI command (e.g., `cch gemini-hook`) that:
- Reads stdin as JSON, uses the Gemini adapter to build a RuleZ Event, and calls hooks::process_event.
- Emits only a Gemini JSON response on stdout; send all logs/errors to stderr.
- On parse errors, log to stderr and return an allow decision (to avoid blocking due to malformed input).
- Avoids exit code 2 for normal denials; rely on Gemini's JSON decision field for enforcement.
  </action>
